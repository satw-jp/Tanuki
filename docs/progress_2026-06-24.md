# Tanuki Plugin 開発進捗 2026-06-24

## プロジェクト概要

- **Tanuki**: Rhino 8 用建築図面自動生成プラグイン（C#, net48, RhinoCommon）
- 平面図・天井伏図・断面図・立面図を3Dモデルから自動生成する
- **リポジトリ**: https://github.com/satw-jp/Tanuki
- **本日のバージョン推移**: v0.5.16 → v0.5.22

---

## 本日実施した変更（カテゴリ別）

### 1. T.Views パネル強化

**課題**: 図面の設定がプロジェクト共通だった。個別図面ごとに設定変更したい。

**実装内容**:
- `ViewDef` に `LayerMode`・`LabelTextHeight` をper-viewフィールドとして追加
- スキーマバージョンを3→4に上げ、移行時にプロジェクト既定値をコピー
- T.Viewsパネルでリスト選択時にマーカー線・インジケーター・描画レイヤー全オブジェクトを一括選択
- **切断高さ編集UI**を追加：平面図・天井伏図のみ有効、断面/立面ではグレーアウト
- Elevationマーカー（インジケーター）を追加

**変更ファイル**: `Data/ViewDef.cs`, `Data/TanukiProject.cs`, `UI/TanukiSectionPanel.cs`, `Commands/AddViewCommand.cs`

---

### 2. 投影プレーン一本化（通り芯ズレ修正）

**課題**: 断面/立面の通り芯・レベル線が大きくズレた位置に描画される。

**原因**: `GenerateSectionOrElevation` 内の各ヘルパー（`AddCrossingGridLines`・`AddLevelLines`・`AddCrossingGridText`・`AddLevelLabels`）がそれぞれ独自に `new Plane(cutPlane.Origin, viewDir)` を作っていた。

**修正**:
```csharp
// ViewGenerator.cs
var cutDirH  = new Vector3d(view.CutEndX - view.CutStartX, view.CutEndY - view.CutStartY, 0);
cutDirH.Unitize();
var projPlane = new Plane(cutPlane.Origin, cutDirH, Vector3d.ZAxis);
var flatten   = Transform.PlaneToPlane(projPlane, Plane.WorldXY);
// 全ヘルパーに projPlane を引数として渡す
```

**変更ファイル**: `Generators/ViewGenerator.cs`

---

### 3. 通り芯ビュー範囲外フィルタ

**課題**: 断面線に交差しない通り芯（例：切断方向と平行に近い通り芯でt=20.8など）が図面の遥か外側に描画される。

**修正**: 交差パラメータ `t` が `[-0.1, 1.1]` 範囲外の通り芯をスキップ

```csharp
double t = (deltaX * gl.DirectionY - deltaY * gl.DirectionX) / cross;
if (t < -0.1 || t > 1.1) continue;  // ビュー範囲外をスキップ
```

**変更ファイル**: `Generators/ViewGenerator.cs`（`AddCrossingGridLines` / `AddCrossingGridText`）

---

### 4. レイヤー構造の整理

**課題**: 通り芯・レベル線が「見え掛かり」「隠れ線」レイヤーに混入していた。

**実装**:
- `LineType` enum に `Level`・`Grid` を追加
- `DrawingPlacer` に「レベル線」（青）・「通り芯」（緑）サブレイヤーを追加
- `AddLevelLines` → `LineType.Level`
- `AddCrossingGridLines` → `LineType.Grid`
- `GridSymbolGenerator.GenerateSymbols` → `LineType.Visible` から `LineType.Grid` に変更
- OriginalLayerモードでもLevel/Gridが正しいサブレイヤーに入るよう対応

**最終的なレイヤー構造**:
```
Tanuki::
  {図面名}::
    断面線   (赤)
    見え掛かり (黒)
    隠れ線   (グレー)
    レベル線  (青 #0078C8)
    通り芯   (緑 #00A050)
    ポシェ   (黒/Solidハッチ)
```

### 5. ポシェ（断面ハッチ）

**課題**: 断面で切断された部分（壁・床など）を塗りつぶしたい。

**実装**: `AddPoché` メソッドを `ViewGenerator` に追加
1. `LineType.Cut` 曲線を収集
2. `Curve.JoinCurves` で閉じたループに結合
3. `Hatch.Create` でSolidハッチを生成
4. 「ポシェ」専用サブレイヤーに配置

断面/立面・平面図/天井伏図の両方に対応。

---

### 6. 平面図 見え掛かり線の追加

**課題**: 平面図でカット高さ以下のオブジェクト（床・低い壁など）が表示されない。

**原因**: `ClassifyFloorPlan` 内で `if (!reflected) continue;` により平面図の見え掛かり処理を完全スキップしていた。

**修正**: 平面図用に `ProjectEdgesDown` ヘルパーを追加（XY平面への真下投影）

```csharp
if (reflected)
{
    // 天井伏図: 背面除去ありの投影（既存）
    ClassifyEdges(obj.Geometry, cutPlane, rcpViewDir, visible, hidden);
}
else
{
    // 平面図: 真下（-Z）投影
    var flatPlane = new Plane(new Point3d(0, 0, cutHeight), Vector3d.XAxis, Vector3d.YAxis);
    ProjectEdgesDown(obj.Geometry, flatPlane, visible);
}
```

### 7. パイプ等の見え掛かり修正

**課題**: 円形パイプ（Extrusion）が見え掛かりレイヤーに出力されない。

**原因**: `ClassifyBrepEdges` の正面判定で、水平キャップ面（法線=Z）は水平なviewDirと内積=0になり `< 0` を満たさないため非正面扱いになっていた。

**修正**:
```csharp
// 変更前
frontFace[fi] = (n * viewDir) < 0.0;
// 変更後（内積=0 の面＝視線に直角な面も正面扱い）
frontFace[fi] = (n * viewDir) <= 0.0;
```

---

### 8. RCP（天井伏図）隠れ線バグ修正

**課題**: 天井伏図で隠れ線が生成されない。

**原因**: `ClassifyFloorPlan` 内でRCP処理時に `hidden` リストに入れたカーブを `result` に追加していなかった。

**修正**: `hidden` も `LineType.Hidden` として result に追加。

---

### 9. デプロイ改善

**課題**: 毎回Program Filesへのコピーが手間。Rhino再起動も手間。

**実装**:
1. **レジストリ変更**: `HKCU:\Software\McNeel\Rhinoceros\8.0\Plug-ins\{GUID}\PlugIn\FileName` をビルド出力先に直接向ける → Program Filesへのコピー不要
2. **`TanukiDevReload` コマンド追加**: 保存→Rhino終了をワンコマンドで実行
   ```csharp
   RhinoApp.RunScript("_Save", false);
   System.Threading.Tasks.Task.Delay(300).ContinueWith(_ =>
       RhinoApp.InvokeOnUiThread(new Action(() =>
           RhinoApp.RunScript("_-Exit", false))));
   ```
3. **`dev-reload.ps1` 作成**: Rhino終了を検知→ビルド→mcpstart付き自動再起動

---

### 10. コードリファクタリング（v0.5.22）

**課題**: `GetOrCreateLayer` が4箇所に別実装、`Replace("::", "_")` が18箇所散在、再帰レイヤー処理が5箇所コピペ。

**実装**: `Generators/LayerUtil.cs` を新規作成して共通化

```csharp
public static class LayerUtil
{
    public static string Safe(string name) => name.Replace("::", "_");

    public static int GetOrCreate(RhinoDoc doc, string name, int parentIdx, Color color) { ... }

    public static void ForEachObject(RhinoDoc doc, int layerIdx,
        System.Action<RhinoObject> action) { ... }  // 再帰
}
```

**置き換え対象**:
- `GetOrCreate`: DrawingPlacer / GridLineDrawer / LevelDrawer / DimensionGenerator / TitleBlockCommand
- `ForEachObject`: ExportCommand / SheetCommands / TanukiSectionPanel / ViewGenerator

## 技術スタック・制約

- **言語**: C# 7.3（records・switch式・null合体演算子 禁止）

- **サニタイズ**: ユーザー入力は全箇所で `.Replace("::", "_")` 必須
